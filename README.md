# PnG LLM Brand Tracker

Инструментарий для исследования того, как **Яндекс Нейро** отвечает на запросы, связанные с брендами **P&G** — и как
бренды P&G выглядят на фоне конкурентов в нейросетевой выдаче.

---

## Пайплайн

```
Яндекс Нейро (браузер)
        │
        │  [1] page_parser.js  ← букмарклет
        ▼
  .txt-файлы (in/)
        │
        │  [2] data_to_sheet.py
        ▼
  out/result.csv
        │
        │  [3] api_analysis.py  ← Mistral AI
        ▼
  out/llm_result.csv
        │
        │  [4] analisys.ipynb
        ▼
  Интерактивные дашборды
```

---

## Структура проекта

```
PnG_caseChamp/
├── page_parser.js       # Букмарклет для сбора ответов Яндекс Нейро
├── data_to_sheet.py     # Парсер txt → CSV / XLSX / JSON
├── api_analysis.py      # LLM-разметка через Mistral AI
├── analisys.ipynb       # Визуализация метрик брендов
├── requirements.txt     # Зависимости
├── in/                  # Входные txt-файлы (структура папок = метаданные)
│   └── {категория}_{бренд}_{тип_вопроса}/
│       └── *.txt
└── out/                 # Результаты
    ├── result.csv
    └── llm_result.csv
```

### Именование папок

Имя папки кодирует метаданные через `_`:

```
гигиена_Pampers_брендовые
│         │       └── категория вопроса
│         └────────── бренд
└──────────────────── категория товара
```

## Запуск

### Шаг 1 — Сбор данных (букмарклет)

1. Открыть `page_parser.js`, скопировать весь код
2. Создать закладку в браузере, вставить код в поле URL
3. Перейти на страницу **Яндекс Нейро** с нужным запросом
4. Нажать закладку → скачается `.txt`-файл с вопросом, ответом и источниками
5. Положить файл в соответствующую папку `in/{категория}_{бренд}_{тип}/`

**Типы вопросов:** `брендовые`, `категориальные`, `консультационные`, `сравнительные`

---

### Шаг 2 — Парсинг txt → таблица

```powershell
# CSV
py data_to_sheet.py --input in --output out/result.csv

# XLSX
py data_to_sheet.py --input in --output out/result.xlsx

# JSON
py data_to_sheet.py --input in --output out/result.json
```

**Выходные поля:**

| Поле                         | Описание                |
|------------------------------|-------------------------|
| `категория_товара`           | Из имени папки          |
| `бренд`                      | Из имени папки          |
| `категория_вопроса`          | Из имени папки          |
| `запрос`                     | Первая строка txt-файла |
| `текст_ответа`               | Тело ответа нейросети   |
| `ссылки_в_тексте`            | URL из текста ответа    |
| `сайты_в_тексте`             | Домены из текста        |
| `ссылки_из_блока_источников` | URL из блока источников |
| `сайты_из_блока_источников`  | Домены из источников    |

---

### Шаг 3 — LLM-разметка через Mistral AI

```powershell
py api_analysis.py
```

Читает `out/result.csv`, отправляет каждую строку в **Mistral Large**, получает структурированную оценку и сохраняет в
`out/llm_result.csv`.

**Вычисляемые метрики:**

| Метрика             | Описание                                               |
|---------------------|--------------------------------------------------------|
| `a_pg_rank`         | Позиция бренда P&G в ответе (0 = не упомянут)          |
| `a_is_winner`       | Назван ли бренд лучшим/рекомендованным                 |
| `a_comp_count`      | Кол-во упомянутых конкурентов (не P&G)                 |
| `a_main_competitor` | Главный конкурент в ответе                             |
| `a_comp_is_winner`  | Победил ли конкурент в рекомендации                    |
| `a_sentiment`       | Тональность: `1` позитив / `0` нейтраль / `-1` негатив |
| `a_has_tech`        | Упомянуты ли технологии/состав                         |
| `a_has_price`       | Упомянуты ли цены/места продаж                         |
| `a_buy_cta`         | Есть ли призыв к покупке                               |
| `a_source_trust`    | Тип доминирующих источников                            |

> API-ключи Mistral задаются в переменной `API_KEYS` в `api_analysis.py`

---

### Шаг 4 — Визуализация (Jupyter)

Открыть `analisys.ipynb` и запустить все ячейки.

**Дашборды:**

- **Позиции брендов** — на каком месте бренд появляется по типам запросов
- **Win Rate** — кто чаще получает рекомендацию: P&G или конкурент
- **Профиль контента** — глубина технологий vs коммерческие атрибуты
- **Sentiment Balance** — тональность ответов по брендам и категориям
- **Угоняющие запросы** — брендовые запросы, где побеждает конкурент
- **Карта источников** — откуда нейросеть берёт информацию о брендах
- **Сводный дашборд** — все ключевые метрики в одной таблице
